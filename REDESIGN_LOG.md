# UniConnect — Redesign Log

A running journal of the editorial-grade frontend rebuild. One paragraph per phase.

---

## Phase 1 — Foundations (complete)

Cut the four-theme system (linen, midnight, daylight, monochrome) down to two — Light and Dark — and rebuilt the token layer from scratch. The new palette is paper-cream + editorial ink for Light, warm charcoal + lifted moss for Dark, with a single deep-moss primary, honey-saffron accent, and a six-stop hue palette (`--hue-a..f`) for places that legitimately need multi-color tags (university chips, avatars, syntax highlight). Swapped the display serif from Playfair to Instrument Serif (Google Fonts) for a sharper, more contemporary editorial voice; Geist Sans + Geist Mono remain. Rewrote `globals.css` from scratch: token blocks for two themes, motion tokens (`--ease-out-soft`, `--dur-base`, etc.), refined scrollbar/selection/focus styles, and a full animation library (marquee, aurora drift, tilt-glow, scroll progress, page fade-up, stagger reveal, link-grow, word-rise, skeleton shimmer, AI caret, ripple, confetti). Migrated every hardcoded hex inside the `coding-*` and `pk-*` (syntax highlight) classes into token references — the entire stylesheet now lives off CSS variables, so future palette tweaks happen in one place. `ThemeProvider` now does legacy migration (old `linen`/`midnight` reads remap to `light`/`dark`), respects OS preference on first visit, and exposes a `toggle()` helper. Added an inline FOUC-prevention script to `app/layout.tsx` so the chosen theme is applied before paint, removing the visibility-hidden flash.

## Phase 2 — Primitives (complete)

Rebuilt `Button` from the ground up: seven variants (`primary`, `accent`, `secondary`, `outline`, `ghost`, `link`, `destructive`) × five sizes × two shapes (`square`, `pill`). The hover state now uses layered shadow + 1px lift instead of the old "brightness 110" trick (which always looks cheap on light themes); active-press is `scale(0.98) translateY(1px)`; ripple is preserved. Added four new primitives the rest of the redesign builds on: `Surface` (token-driven card with `tone`, `elevation`, `radius`, and `interactive` variants), `Pill` (badge with the full hue-palette tone set baked in, used everywhere multi-color tags appear), `Field` (input wrapper with label/hint/error/prefix/suffix and a focus ring that draws a 4px primary halo), and `Kbd` (keyboard chip for the upcoming command palette). All five components live entirely on tokens — there is no raw color anywhere in `components/ui/`.

## Phase 3 — Marketing chrome (complete)

Replaced the four-theme dropdown switcher with a true two-state `ThemeSwitcher` — a 36px circular toggle that crossfades sun ↔ moon icons with a 90° rotate (Framer `AnimatePresence`, `mode="wait"`). It is a drop-in (same export name) so the rest of the codebase keeps working unchanged. Rebuilt `MarketingNav` as a floating glass pill that lives 12px from the top of the viewport: it has a translucent background that only materializes after 24px of scroll, the center nav links sit inside a sunken pill-within-a-pill ("Features · Universities · Library · Careers · AI" — renamed Notes→Library and Jobs→Careers for the editorial voice), and the new `Logomark` is a black ink disc with a saffron dot, paired with the Instrument Serif wordmark. CTAs use the new pill-shape primary button. Mobile drawer animates in as a separate floating card with the same radius vocabulary. Footer rebuilt as a magazine masthead: a hero callout in 8vw display serif ("Made for the *student*, by the *student*."), eyebrow micro-mono section labels (`eyebrow` class), 4-column link grid with `link-grow` underlines, and a colophon strip with the technology stack in monospace. Everything sits on the new token palette and respects both themes.

## Phase 7 — Cleanup (complete)

Final sweep across the codebase. Removed every reference to the old four-theme system from `lib/utils.ts`, `ThemeProvider`, `app/layout.tsx`, and the appearance settings page — the only remaining occurrences of the old theme names live in the legacy-migration branch (so existing user storage maps cleanly to the new system) and in unrelated copy ("Resets at midnight."). Rewrote `app/(app)/profile/settings/page.tsx` from scratch on the new design system: editorial header, `Surface`-wrapped sections, `Field`/`Pill`/`Button` primitives throughout, and a 2-card theme picker (Light/Dark) with token-driven swatches that visually match the actual themes — replacing the old 4-card grid with hardcoded hex swatches. Replaced raw hex colors with token references across the dashboard surface area: `feed/page.tsx` (StatCard hues + hero gradient), `study/page.tsx` (timer mode colors, preset palette, podium gradients via Tailwind utilities), `study/analytics/page.tsx`, `calendar/page.tsx`, `gpa/page.tsx` (grade color thresholds), `universities/page.tsx` and `[slug]/page.tsx` (rating star fill → `--warning`), `chat/[channelId]/page.tsx` (attachment menu colors → `--hue-*`), and `coding/layout.tsx` (full rewrite using `Pill` and the token system). The coding sub-system (`coding/practice`, `coding/dry-runs`, `coding/visual-labs`) preserves its purposefully distinct "Purple Knight" code-editor palette via the `.pk-*` and `coding-*` classes in `globals.css` — those classes themselves are now token-driven (mapped to the hue palette), so the editor still reads like Dracula but recolours under the Light theme automatically.

## Phase 6 — Auth flows (complete)

Rewrote the auth surface as a split-screen editorial: the form lives in the left column, the right column is a magazine cover panel with an aurora-mesh backdrop, an oversized italic 140px serif quotation mark, a 3.4vw display-serif pull quote attributed to a real student archetype (NUST · SEECS), a hairline-rule attribution block, and a typographic university roster ("Across 12+ universities") in dictionary-style listing. The header carries the editorial Logomark + "Issue №01 — 2026" mono marker, and the footer is a colophon strip. **Login** opens with an "Returning student" eyebrow and a 52px serif "Welcome back." headline; the form uses the new `Field` primitive with `prefix`/`suffix` adornments and the focus-halo ring. **Signup** mirrors the same shape ("Create your *account.*") with inline username availability — the suffix slot now shows the spinner / check / alert icon natively via `Field`'s `suffix` prop, and the `hint`/`error` slots replace the old hand-rolled status text. Submitted-state shows a saffron `MailCheck` motif card with "Check your inbox." Resend cooldown preserved. **Forgot password** ("Forgot your *password?*") and **Reset password** ("Set a new *password.*" with four pill-shaped strength chips that toggle to positive-tinted on satisfy) follow the same editorial template. **Verify** has three states: an opening loader card with serif "Verifying your email…", a positive-tinted success card with bouncing dot indicator that auto-routes to onboarding, and an error card with a serif "That link is *stale.*" and a resend CTA. **Onboarding** keeps its three-step wizard logic — university search, branch/department/year selection, avatar + bio — but the surface is rebuilt: editorial step indicator with mono numerals (`01 — 03`), per-step display-serif headlines that change ("Pick your *campus.*" → "Tell us your *details.*" → "Make it *yours.*"), pill-shape year buttons that go inverted-ink when selected, university list rows with mono short-name chips and `CheckCircle` selection, and a refined avatar uploader with side-info column. Forms use the `Field` primitive throughout and respect `--field` / `--field-line` tokens for the focus halo.

## Phase 5 — App shell (complete)

Refreshed `Sidebar` and `Topbar` so the dashboard reads as the same magazine the marketing pages just opened. The sidebar now uses the editorial Logomark (ink disc + saffron dot + Instrument Serif wordmark), `eyebrow`-class section labels (`Workspace`), tighter `261px` width with `border-line` rather than the heavier `border-r`, and the active-nav indicator (`layoutId="active-nav"`) animates as a soft moss pill behind the active item — no more gradient mush. Expandable sub-groups (Communication, Career Center, Analytics) animate via Framer with a left-rule that visually anchors the children to the parent. The `NEW` flag is now a `Pill tone="primary" size="xs"` instead of a hand-rolled emerald chip, so it inherits the moss palette under both themes. The mobile drawer keeps its spring-out animation but now uses a subtle backdrop-blur scrim. The `Topbar` becomes the dashboard's command-palette surface: search input is now a 40px pill with ⌘K kbd hint on the right, focus draws a 4px primary halo, and the dropdown beneath it is a magazine-style list — token-tinted `Pill`s for note/job/university/user types (no more hardcoded blue/green/violet utility classes), monospace timestamps, and a kbd hint footer ("Press Enter to open library · Esc"). Greeting text uses Instrument-Serif tracking and a monospaced sub-line with notification count. Notifications dropdown got an editorial empty state ("All clear.") and quiet `✦` glyphs instead of emoji. The "New post" button is the inverted ink pill — high contrast, restrained.

## Phase 4 — Landing (complete)

Rebuilt every section of the marketing homepage as a single coherent magazine spread. The **Hero** now opens with an issue-marker eyebrow ("Issue №01 — A campus, in your pocket") and a "Built for Pakistani students" pulse-pill, then drops a 11.5vw display-serif headline ("One campus. *Every* campus.") with `TextReveal` per-word stagger; the right column carries a four-line subline, primary/ghost pill CTAs, and a refined six-avatar social-proof stack drawn from the new hue tokens (no hardcoded hex anywhere). Below the fold, three vignette `Surface` cards preview the product — a chat thread (university-tagged `Pill`s), a library shelf (token-tinted note covers), and an AI Q&A with the streaming caret. **UniversityTicker** is now a quiet hairline-bordered marquee in 3.4vw italic Instrument Serif with edge-fade gradients on both sides — no more colored chips. **FeaturesSection** is a 12-column bento with two wide cards and four narrow ones, each indexed (№01..06), each with its own hue accent, and each carrying its own bespoke vignette (chat lines, library tiles, careers card, AI exchange, mastery percentage, tokenized syntax-highlighted code). **StatsSection** dropped the gradient-text trick in favor of huge 9vw display numerals with token-grey suffixes — pure typographic confidence. **HowItWorks** is now a vertical timeline with a scroll-linked gradient line that fills as the user passes through; three "Step 0X" entries in display serif. **TestimonialsSection** uses pull-quote cards with an oversized italic "&ldquo;" mark and a hairline footer strip carrying initials + university in monospace. **FAQSection** is a numbered editorial list (№ 01..06) with a `+`-icon expand affordance that rotates 45° on open. **CTASection** closes with an 11vw display serif "Ready to *connect?*" over an aurora-mesh, two pill CTAs, and a tiny mono epigram ("NO CREDIT CARD · NO SPAM · NO BS"). The root `app/page.tsx` now wires `ScrollProgress` and `MagneticCursor` so the homepage participates in the same motion vocabulary as the rest of the marketing surface.
