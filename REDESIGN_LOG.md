# UniConnect ‚Äî Redesign Log

A running journal of the editorial-grade frontend rebuild. One paragraph per phase.

## Phase 1 ‚Äî Foundations (complete)

Rebuilt theming as a strict **two-mode system** (Light default + Dark) and wired it end-to-end via CSS variables and `data-theme`. Token blocks live in `globals.css`, and UI references tokens only (no raw hex in components). Added an inline FOUC-prevention script in `app/layout.tsx` so the chosen theme is applied before first paint, and updated the theme provider to persist choice in localStorage while respecting `prefers-color-scheme` on first visit. Typography is loaded via `next/font` and exposed through CSS variables for consistent use across Tailwind utilities.

## Phase 2 ‚Äî Primitives (complete)

Rebuilt `Button` from the ground up: seven variants (`primary`, `accent`, `secondary`, `outline`, `ghost`, `link`, `destructive`) √ó five sizes √ó two shapes (`square`, `pill`). The hover state now uses layered shadow + 1px lift instead of the old "brightness 110" trick (which always looks cheap on light themes); active-press is `scale(0.98) translateY(1px)`; ripple is preserved. Added four new primitives the rest of the redesign builds on: `Surface` (token-driven card with `tone`, `elevation`, `radius`, and `interactive` variants), `Pill` (badge with the full hue-palette tone set baked in, used everywhere multi-color tags appear), `Field` (input wrapper with label/hint/error/prefix/suffix and a focus ring that draws a 4px primary halo), and `Kbd` (keyboard chip for the upcoming command palette). All five components live entirely on tokens ‚Äî there is no raw color anywhere in `components/ui/`.
